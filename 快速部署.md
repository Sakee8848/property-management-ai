## ⚡ 最快方案：GitHub Pages + MVP原型

### ✅ 优点
- 完全免费
- 3分钟完成
- 无需注册额外服务
- 直接可用

### 📋 步骤

#### 1️⃣ 创建GitHub仓库（2分钟）

访问：https://github.com/new

填写信息：
```
Repository name: property-management-ai
Description: 智能物业管理AI系统演示
Public (选择公开)
不要勾选：Initialize this repository with a README
```

点击 "Create repository"

#### 2️⃣ 推送代码（1分钟）

复制下面的命令（替换YOUR_USERNAME为你的GitHub用户名）：

```bash
cd /Users/tonyyu/Documents/物业管理

# 添加远程仓库
git remote add origin https://github.com/YOUR_USERNAME/property-management-ai.git

# 推送到main分支
git branch -M main
git push -u origin main

# 创建并推送gh-pages分支
git checkout -b gh-pages
git push origin gh-pages
git checkout main
```

#### 3️⃣ 启用GitHub Pages（30秒）

1. 进入仓库页面
2. 点击 **Settings** 标签
3. 左侧点击 **Pages**
4. Source 选择：`gh-pages` 分支
5. 点击 **Save**

等待30秒...

#### 🎉 完成！获得访问地址

**MVP原型地址**：
```
https://YOUR_USERNAME.github.io/property-management-ai/mvp-prototype.html
```

**测试版地址**：
```
https://YOUR_USERNAME.github.io/property-management-ai/frontend/test-standalone.html
```

## 🌟 方案二：Vercel部署（推荐，功能最完整）

### 优势
- ✅ 完整的Vue应用
- ✅ 自动部署
- ✅ 全球CDN加速
- ✅ 自定义域名

### 快速步骤

#### 1️⃣ 注册Vercel

访问：https://vercel.com/signup

- 选择 "Continue with GitHub"
- 授权Vercel访问GitHub

#### 2️⃣ 导入项目

1. 点击 "Add New..." → "Project"
2. 找到 `property-management-ai` 仓库
3. 点击 "Import"

#### 3️⃣ 配置项目

```
Framework Preset: Vite
Root Directory: frontend
Build Command: npm run build
Output Directory: dist
Install Command: npm install
```

#### 4️⃣ 环境变量（可选）

添加环境变量：
```
VITE_USE_MOCK = true
```

#### 5️⃣ 部署

点击 "Deploy" 按钮

等待2-3分钟...

#### 🎉 完成！

获得URL：`https://your-project.vercel.app`

## 🔧 常见问题

### Q: 部署后页面空白？

**A**: 检查以下几点：

1. 确认build命令正确
```json
{
  "scripts": {
    "build": "vite build"
  }
}
```

2. 修改 `frontend/vite.config.js`：
```javascript
export default defineConfig({
  base: './',  // 添加这行
  // ... 其他配置
})
```

3. 清除缓存重新构建

### Q: 404错误？

**A**: 配置路由重定向

Vercel: 已配置 `vercel.json`
Netlify: 已配置 `netlify.toml`

### Q: 手机上显示异常？

**A**: 检查viewport设置

```html
<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">
```

### Q: 如何更新部署？

**A**: 

```bash
cd /Users/tonyyu/Documents/物业管理

# 修改代码后
git add .
git commit -m "更新内容"
git push

# Vercel/Netlify会自动重新部署
# GitHub Pages需要重新推送gh-pages分支
```
